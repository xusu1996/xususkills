# 内容真实性规则

## 目标

让图片看起来像真实对话记录，同时适合公开发布。

## 允许的处理

- 删除不影响上下文的寒暄、重复、工具噪音。
- 用 `...` 标记省略内容。
- 调整换行。
- 按消息边界分页。
- 遮盖隐私信息。
- 为卡片增加简短标题、日期、页码或“节选”标记。

## 禁止的处理

- 改写用户或 AI 的原话。
- 添加未发生的问答。
- 把原本不相邻的多段对话伪装成连续上下文。
- 把谨慎表达改成确定表达。
- 把普通解释包装成夸张结论。
- 为了传播效果添加标题党、金句或行动号召。

## 隐私检查

默认遮盖：

- 本地绝对路径
- 真实姓名
- 电话、邮箱、微信号、账号 ID
- API key、token、密码
- 订单号、身份证号、银行卡号
- 私密地点和具体住址

路径可以改成：

```text
[本地路径已隐藏]
```

密钥和账号可以改成：

```text
[已隐藏]
```

## 图片隐私

- 图片路径只用于读取素材，不显示在卡片上。
- 图片本身可能包含头像、昵称、账号、聊天内容、定位、订单号等隐私。
- 生成含图片的卡片后，提醒用户检查图片内容是否可以公开发布。
- 不对图片内容做自动涂抹或裁剪，除非用户明确要求。

## 省略标记

省略中间内容时使用：

```text
...
```

不要使用“此处省略很多精彩内容”这类带评价的文字。
